# 스마트 팩토리 자동화 프로젝트

## 1. 서론 (Introduction)


### 연구 배경 및 목적
- 공장에서 물건을 만들 때 보통 'PLC'라는 전용 컴퓨터를 쓰는데, 이 프로젝트에서는 우리가 평소에 쓰는 프로그래밍 언어(파이썬, Python)와 산업용 통신(Modbus TCP)을 연결해서 공장을 똑똑하게 제어해 보았습니다.

- Factory I/O라는 가상의 공장 시뮬레이터 프로그램을 사용했습니다. 물건을 깎는 기계(가공기), 물건을 실어 나르는 컨베이어 벨트, 물건을 툭 밀어내는 밀대(푸셔), 그리고 눈에 보이지 않게 물건을 감지하는 센서들이 잘 움직이도록 제어 프로그램을 만들었습니다.


### 연구 가설 (Research Hypothesis)
- 가설 1 (일꾼을 여럿 두기): 한 가지 일만 순서대로 하는 것보다, 여러 일을 동시에 처리할 수 있게 일꾼(Thread)을 나누고 물건 대기열(Queue)을 만들면 공장이 멈추지 않고 훨씬 매끄럽게 돌아갈 갈 것입니다.

- 가설 2 (새치기 방지하기): 여러 일꾼이 동시에 컴퓨터(통신)에 접속해 데이터를 바꾸려고 하면 충돌이 날 수 있으니, 출입증(Lock)을 줘서 한 번에 한 명씩만 안전하게 일하게 통제하면 오류가 안 생길 것입니다.


### 핵심 구현 목표

- 파이썬 프로그램으로 공장 기계의 스위치를 켜고 끄며 상태를 실시간으로 확인.


- 일꾼을 나누고 대기열을 만들어 물건을 척척 분류.




## 2. 시스템 구성 및 통신 아키텍처 (System Architecture & Communication)

### 네트워크 및 통신 환경

- 통신 규격: Modbus TCP 기반의 산업용 네트워크 통신 환경 구축 (IP: `210.119.14.58`, Port: `502`, Unit ID: `1`).

- 통신 중에 인터넷이 잠깐 끊기거나 지연되더라도 프로그램이 멈추지 않고 다시 연결(Reconnection)을 시도하도록 안전장치를 넣었습니다.


### 시스템 데이터 흐름 아키텍처 (Data Flow Architecture)

- 구조적 흐름: Python Control Script ⇄ Modbus TCP Socket Layer ⇄ Factory I/O Virtual PLC/Plant 환경 간의 양방향 실시간 데이터 교환 구조 확립.

- 상위 제어기(Python)가 폴링(Polling) 주기에 따라 센서 입력 상태를 읽어오고(Read Discrete Inputs), 연산된 제어 명령을 하위 액추에이터와 공정에 코일(Write Coils) 형태로 즉각 송신하는 이벤트-루프 기반 인터페이스 아키텍처 설계.


### I/O 매핑 설계

- 입력받는 것 (센서): 기계가 바쁘게 일하는지(`BUSY`), 에러가 났는지(`ERROR`), 물건이 어디쯤 왔는지 감지하는 센서들입니다.

- 내보내는 것 (동작): 물건을 뱉어내는 장치(에미터), 기계 동작 버튼, 물건을 옆으로 미는 밀대(푸셔), 컨베이어 벨트들을 움직이는 신호입니다.





## 3. 핵심 제어 알고리즘 및 구현 (Core Control Algorithms)

### 생산 및 가공 공정 제어 (Machining Center Control)
- `MC0`(블루 자재 전용)와 `MC1`(그린 자재 전용)의 상태 전이(Idle -> Busy -> Done)를 감지하는 폴링 및 타임아웃 관리 함수 구현했습니다.


- 기계가 일할 준비가 되었는지, 지금 일하고 있는지, 끝났는지를 계속 확인하고 혹시 기계가 고장(에러) 나면 알아서 초기화 버튼을 눌러 다시 시작하게 만들었습니다.


### 스레드 기반 비동기 큐 시스템 (Thread & Queue-based Sorting)
- 일꾼 나누기: 물건을 공장에 넣는 일꾼(production_loop)과 물건이 지나갈 때 밀대로 툭 쳐서 분류하는 일꾼(sorting_loop)을 따로따로 두어 두 일이 동시에 척척 진행되게 했습니다.


- 줄 세우기 (Queue): 물건이 너무 한꺼번에 오면 복잡해지니까 최대 4개까지만 줄을 세워(MAX_QUEUE_SIZE = 4) 순서가 뒤바뀌지 않게 관리했습니다.


- 딱 맞는 타이밍 맞추기: 물건이 정확히 밀대 앞에 도착했을 때 딱 맞춰서 신호를 주는 기술(threading.Event)을 적용했습니다.


### 스레드 동기화 및 락(Lock) 메커니즘 (Concurrency & Lock Management)
- 멀티스레딩 환경에서 공유 자원(Modbus 레지스터 I/O 데이터 및 공유 큐 구조체)에 대한 동시 접근으로 인해 발생할 수 있는 경쟁 상태(Race Condition) 및 데이터 오염 방지합니다.

- `threading.Lock` 객체를 도입하여 Modbus 통신 데이터 읽기/쓰기 구간에 임계 구역(Critical Section)을 설정, 멀티스레드 간 안전한(Thread-safe) 통신 및 제어 무결성 확보합니다.


## 4. 시스템 진단 및 예외 처리 (Diagnostics & Exception Handling)

- 실시간 모니터링: 센서가 잘 작동하는지 눈으로 쉽게 볼 수 있는 감시 기능(`monitor_sort_area`)을 넣었습니다.


- 미리 방지하는 에러 처리: 통신이 늦어지거나 대기줄이 꽉 차서 물건이 건너뛰어져도(`Queue full`, `skipped`) 프로그램이 튕기지 않고 로그를 남기며 의연하게 대처합니다.

- 부품별 테스트: 밀대만 따로 움직여 보거나 기계를 강제로 시험 가동해 보는 기능(`test_pusher_only`, `force_mc0_test`)을 만들어 어디가 고장 났는지 바로바로 확인할 수 있습니다.


## 5. 결론 및 향후 개선 방안 (Conclusion)

### 성과 요약
- 파이썬과 가상 공장을 무선 통신으로 튼튼하게 엮어서, 파란색과 초록색 물건이 섞여 와도 알아서 척척 분류하는 스마트 팩토리 공정을 성공적으로 완성했습니다.


- 일꾼을 나누고(스레드) 줄을 세우고(큐) 자물쇠를 채우는(락) 기술을 써서, 공장이 멈추지 않고 부드럽고 빠르게 돌아가도록 만들었습니다.


### 한계점 및 향후 과제
- 가상 시뮬레이터 환경을 넘어 실제 산업 현장의 하드웨어 PLC 및 분산 제어 시스템(DCS) 환경과의 연동성 확장할 것 입니다.

- 네트워크 단절이나 센서 오작동 등 극한의 예외 상황 발생 시 자동 복구(Auto-recovery) 및 폴백(Fallback) 알고리즘 고도화할 것 입니다.
